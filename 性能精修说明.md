# 性能精修说明

## 1. 热路径范围
- 热路径 A（分页查询）：`BusinessTaskReadService` → `BusinessTaskRepository.CountByQueryConditionsAsync/QueryByQueryConditionsAsync`
- 热路径 B（看板聚合）：`GlobalDashboardQueryService`、`DockDashboardQueryService`、`SortingReportQueryService` → 仓储聚合
- 热路径 C（扫描主链路）：`ScanIngressService`、`TaskExecutionService`、`ScanMatchService`
- 热路径 D（回写链路）：`WmsFeedbackService`、`OracleWmsFeedbackGateway`、`FeedbackCompensationService`

## 2. 本次“进一步精修”项（区别于主路径收口）

### 2.1 跨分表查询路径精修
- 保留按时间范围计算命中分表的主路径，避免不在时间窗内的分表扫描。
- 查询过滤从 `Trim()` 计算式改为可走索引的等值判断，减少函数包裹导致的索引失效风险。
- 码头与波次空值匹配改为显式 `null/空字符串` 分支，减少热路径字符串函数调用。

### 2.2 分页总数统计与分页读取精修
- `CountByQueryConditionsAsync`、`QueryByQueryConditionsAsync` 继续保持仓储侧下推过滤、排序、分页。
- 大偏移分页场景保持“按分片逐段跳过”策略，避免先全量取回再分页。
- 分页排序字段补充 `(CreatedTimeLocal, Id)` 组合索引，降低倒序分页退化概率。

### 2.3 看板/报表聚合精修
- 聚合继续在仓储侧执行，保持 SQL 侧 `GroupBy` 聚合，避免应用层全量内存聚合。
- 聚合条件相关字段补充组合索引：`CreatedTimeLocal + SourceType + Status + IsException + IsRecirculated`、`WaveCode + CreatedTimeLocal`、`CreatedTimeLocal + WaveCode + TargetChuteCode + ActualChuteCode`。
- 去除聚合路径内不必要 `Trim()`，降低跨分表循环中的表达式开销。

### 2.4 扫描与匹配链路精修
- `TaskExecutionService` 增加请求字符串一次归一化，复用条码/设备号/TraceId，减少重复 `Trim()` 与重复赋值。
- `ScanMatchService` 改为一次归一化后查询，匹配失败消息使用归一化值，避免重复处理。
- 扫描日志索引补充 `TaskCode`、`(Barcode, ScanTimeLocal)`、`(TaskCode, ScanTimeLocal)`，提高链路排查查询速度。

### 2.5 回写与补偿链路精修
- `OracleWmsFeedbackGateway` 保持数组绑定批量写入，按来源类型分组后单连接批量执行。
- `FeedbackCompensationService` 在整批失败时不再重复执行“失败→失败”状态更新，减少无效写入。
- 回写补偿查询字段补充索引：`FeedbackStatus`、`IsFeedbackReported`、`FeedbackTimeLocal` 与组合索引 `FeedbackStatus + IsFeedbackReported + FeedbackTimeLocal`。

## 3. 已完成精修项清单
- [x] 过滤表达式去函数化（去除热路径 `Trim()`）
- [x] 扫描链路字符串归一化复用
- [x] 回写补偿失败分支冗余更新移除
- [x] 业务任务、扫描日志、落格日志、同步批次索引补齐

## 4. 受当前架构限制暂不继续下钻项
- 当前仍为“跨分表循环 + 分片内聚合/分页”的实现，未引入统一物化视图或查询中台。
- 未引入 Keyset Pagination（游标翻页）替代大偏移页码查询，仍使用 `Skip/Take`。
- 未引入回写状态批量更新 SQL（当前仍按任务更新本地状态，便于保持审计语义稳定）。

## 5. 当前性能边界
- 时间范围跨度越大，分表命中数量越多，跨分表循环成本线性上升。
- 超大页码下，仍存在跨分片逐段跳过带来的计数开销。
- 看板聚合在峰值并发下仍依赖分片数量与 SQL Server 实时聚合能力。

## 6. 后续极限优化建议
1. 引入按天/按波次的聚合快照表，降低实时聚合开销。
2. 对大偏移查询引入游标翻页方案（`CreatedTimeLocal + Id`）。
3. 对回写状态更新增加批量更新 SQL，减少逐条更新往返。
4. 增加跨分表命中统计与慢查询观测，支持按月热点分片治理。
