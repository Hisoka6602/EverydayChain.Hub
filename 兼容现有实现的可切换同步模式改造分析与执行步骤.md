# 兼容现有实现的可切换同步模式改造分析与执行步骤

## 1. 目标与边界
- 目标：在不推翻现有同步链路的前提下，引入“可切换同步模式”，同时保留既有 keyed merge 主流程。
- 边界：本次仅新增模式分流能力与对应组件，不重构既有删除治理、分表治理、检查点与调度框架。

## 2. 当前代码现状（基于仓库实现）
- `SyncExecutionService.ExecuteBatchAsync` 当前固定走：Oracle 分页读取 → 暂存 → Upsert 合并 → 删除识别/执行 → 变更日志/删除日志 → 检查点提交。
- `SyncOrchestrator.RunTableSyncAsync` 负责读取配置、检查点、计算窗口并调用执行服务，暂未按同步模式分支。
- `OracleSourceReader.ReadIncrementalPageAsync` 当前按游标窗口分页读，未提供“按状态列 + ROWID 回写协作”能力。
- `SqlServerSyncUpsertRepository.MergeCoreAsync` 当前承担 keyed merge 与 `sync_target_state*` 状态跟踪。
- `SyncCheckpointRepository` 当前负责检查点读写，已是可复用的模式无关基础能力。
- 现有 `SyncMode` 位于 `Domain/Enums/SyncMode.cs`，枚举值为 `InitialFull/Incremental`，与新需求 `KeyedMerge/StatusDriven` 语义不一致。

## 3. 需求对照与实现策略

### 3.1 模式定义与兼容默认值
- 新增同步模式：`KeyedMerge`、`StatusDriven`。
- 老配置兼容：未配置 `SyncMode` 时默认 `KeyedMerge`，确保存量配置不报错。
- 处理建议：统一到单一 `SyncMode` 枚举来源，避免 `Domain/Enums` 与 `Domain/Sync/Enums` 双定义并存。

### 3.2 KeyedMerge 模式（保留现有链路）
- 保留现有 `ExecuteBatchAsync` 主流程（读取、暂存、Merge、删除、日志、检查点）。
- 继续使用 `SqlServerSyncUpsertRepository.MergeCoreAsync` 与 `sync_target_state*`。
- 继续使用唯一键状态加载、缺失删除识别与删除策略执行。

### 3.3 StatusDriven 模式（新增链路）
- 读取条件：按 `StatusColumnName = PendingStatusValue` 读取远端数据。
- 新增兼容要求：`TASKPROCESS` 可能为 `null`，当 `PendingStatusValue` 配置为 `null` 时，读取条件必须切换为 `StatusColumnName IS NULL`（禁止拼接为 `= NULL`）。
- 默认值可配置化：默认 `TASKPROCESS=N`（待消费）、回写默认 `Y`（已完成），但必须可配置覆盖。
- 本地写入策略：仅追加写入（AppendOnly），不做 Upsert、不做 Delete。
- 回写策略：可配置 `ShouldWriteBackRemoteStatus`；开启时将远端状态改为 `CompletedStatusValue`。
- 回写定位：读取时必须额外带出 `ROWID as __RowId`，回写必须优先且仅按 `ROWID` 更新，禁止按主键更新。
- 禁用项（仅在 StatusDriven 下）：`MergeCoreAsync`、`sync_target_state`、缺失删除、唯一键状态表加载。

## 5. 配置扩展点（向后兼容）
- 在同步表配置扩展字段：
  - `SyncMode`
  - `StatusColumnName`
  - `PendingStatusValue`
  - `CompletedStatusValue`
  - `ShouldWriteBackRemoteStatus`
  - `BatchSize`
- 映射路径：`SyncTableOptions` → `SyncTableDefinition`，并在 `SyncTaskConfigRepository` 完成默认值与合法性校验。
- 兼容要求：老配置缺失以上字段时可正常加载，且默认进入 `KeyedMerge`。

## 6. 执行步骤（细化）
1. **统一模式模型**
   - 新建目标 `SyncMode` 枚举与状态消费配置模型。
   - 完成 `SyncTableDefinition` 的模式与状态消费属性扩展。
   - 处理旧枚举迁移，确保仓库内仅保留一套生效定义。
2. **扩展配置绑定与校验**
   - 在 `SyncTableOptions` 增加模式与状态驱动参数，补齐中文注释与可填写范围。
   - 对 `PendingStatusValue` 增加空值语义：允许 `null`，并在运行时映射为 `IS NULL` 查询分支。
   - 在 `SyncTaskConfigRepository` 增加解析、默认值填充与错误信息（中文）校验。
3. **新增 StatusDriven 基础组件**
   - 新增 Oracle 状态驱动读取器（带 `__RowId`，并支持 `PendingStatusValue` 的“等值/空值”双分支过滤）。
   - 新增 SQL Server 仅追加写入器。
   - 新增 Oracle 远端状态回写器（按 `ROWID`）。
   - 新增 `RemoteStatusConsumeService` 串联读取→追加→可选回写，并做异常隔离与日志。
4. **执行服务分流**
   - 在 `SyncExecutionService.ExecuteBatchAsync` 按模式分流：
     - `KeyedMerge`：调用原逻辑。
     - `StatusDriven`：调用 `IRemoteStatusConsumeService`。
   - 确保 KeyedMerge 分支行为不变。
5. **依赖注入与装配**
   - 在 `ServiceCollectionExtensions` 注册新增抽象与实现。
   - 保持分层依赖方向：Host → Infrastructure → Application → Domain。
6. **配置样例与文档联动**
   - 更新 `appsettings.json` 示例，补齐新字段注释与数组示例项。
   - 说明默认行为（不配 `SyncMode` 即 `KeyedMerge`）。
7. **测试与回归**
   - 增加配置映射测试（默认值与兼容性）。
   - 增加 StatusDriven 服务测试（仅追加、可选回写、异常隔离）。
   - 回归 KeyedMerge 既有测试，确保未回归。

## 7. 旧逻辑保留/停用清单（满足最终输出口径）

### 7.1 保留的旧逻辑
- `SyncOrchestrator.RunTableSyncAsync` 的“配置读取 + 检查点 + 窗口计算 + 执行入口”框架保留。
- `SyncExecutionService` 现有 keyed merge 执行主干保留。
- `OracleSourceReader.ReadIncrementalPageAsync` 继续服务 keyed merge 增量读取。
- `SqlServerSyncUpsertRepository.MergeCoreAsync`、`sync_target_state*`、删除识别与执行逻辑在 keyed merge 下继续保留。
- `SyncCheckpointRepository` 持续作为两种模式共用的检查点能力。

### 7.2 仅在 StatusDriven 模式停用的旧逻辑
- `SqlServerSyncUpsertRepository.MergeCoreAsync`。
- `sync_target_state*` 状态持久化与加载。
- 缺失删除链路（删除识别、删除执行）。
- 基于唯一键的状态表加载与差异处理。

## 8. 可删除冗余代码判断
- 若迁移完成后 `Domain/Enums/SyncMode.cs` 与新位置枚举重复，应删除旧定义与相关旧引用，避免双枚举并存导致语义漂移。
- 若存在仅为模式分支做一层透传的新旧方法，应合并并删除透传层，避免新增无意义中间层。
- 除上述重复定义与透传代码外，不建议删除现有 keyed merge 链路代码。

## 9. 待确认项（实施前必须确认）
- `BatchSize` 在 StatusDriven 中是“读取批大小”还是“本地追加批写大小”，或两者分离。
- StatusDriven 是否仍复用现有 `CursorColumn + Window` 边界，还是仅按状态列拉取。
- 远端回写失败策略：整批失败回滚、部分成功重试、还是记录失败留待下轮重试。
- `CompletedStatusValue` 回写时是否需要附带“完成时间/批次号”等审计列更新。
