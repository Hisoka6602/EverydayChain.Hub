# 兼容现有实现的可切换同步模式改造分析与执行步骤

## 1. 目标与边界
- 目标：在不推翻现有同步链路的前提下，引入“可切换同步模式”，同时保留既有 keyed merge 主流程。
- 边界：本次仅新增模式分流能力与对应组件，不重构既有删除治理、分表治理、检查点与调度框架。

## 2. 当前代码现状（基于仓库实现）
- `SyncExecutionService.ExecuteBatchAsync` 已按 `SyncMode` 分流：`KeyedMerge` 走原链路，`StatusDriven` 委托 `IRemoteStatusConsumeService`。
- `SyncOrchestrator.RunTableSyncAsync` 保持“读取配置+检查点+窗口计算+执行入口”框架不变，由执行服务内部处理模式分支。
- `OracleSourceReader.ReadIncrementalPageAsync` 继续服务 `KeyedMerge`；`OracleStatusDrivenSourceReader` 已提供状态读取与 `__RowId` 输出能力。
- `SqlServerSyncUpsertRepository.MergeCoreAsync` 继续承担 `KeyedMerge` 与 `sync_target_state*` 状态跟踪。
- `SyncCheckpointRepository` 继续作为两种模式共用的检查点能力。
- `SyncMode` 已统一位于 `EverydayChain.Hub.Domain/Enums/SyncMode.cs`，枚举值为 `KeyedMerge/StatusDriven`。

## 3. 需求对照与实现策略

### 3.1 模式定义与兼容默认值
- 新增同步模式：`KeyedMerge`、`StatusDriven`。
- 老配置兼容：未配置 `SyncMode` 时默认 `KeyedMerge`，确保存量配置不报错。
- 处理建议：统一到单一 `SyncMode` 枚举来源，避免 `Domain/Enums` 与 `Domain/Sync/Enums` 双定义并存。

### 3.2 KeyedMerge 模式（保留现有链路）
- 保留现有 `ExecuteBatchAsync` 主流程（读取、暂存、Merge、删除、日志、检查点）。
- 继续使用 `SqlServerSyncUpsertRepository.MergeCoreAsync` 与 `sync_target_state*`。
- 继续使用唯一键状态加载、缺失删除识别与删除策略执行。

### 3.3 StatusDriven 模式（新增链路）
- 读取条件：按 `StatusColumnName = PendingStatusValue` 读取远端数据。
- 新增兼容要求：`TASKPROCESS` 可能为 `null`，当 `PendingStatusValue` 配置为 `null` 时，读取条件必须切换为 `StatusColumnName IS NULL`（禁止拼接为 `= NULL`）。
- 默认值可配置化：默认 `TASKPROCESS=N`（待消费）、回写默认 `Y`（已完成），但必须可配置覆盖。
- 本地写入策略：仅追加写入（AppendOnly），不做 Upsert、不做 Delete。
- 回写策略：可配置 `ShouldWriteBackRemoteStatus`；开启时将远端状态改为 `CompletedStatusValue`。
- 回写定位：读取时必须额外带出 `ROWID as __RowId`，回写必须优先且仅按 `ROWID` 更新，禁止按主键更新。
- 禁用项（仅在 StatusDriven 下）：`MergeCoreAsync`、`sync_target_state`、缺失删除、唯一键状态表加载。

## 6.2 方法级职责边界（减少歧义）
- `SyncExecutionService.ExecuteBatchAsync`
  - 只负责“选择执行路径 + 统一批次生命周期管理 + 指标日志”。
  - KeyedMerge 路径复用原有逻辑块，不迁移既有实现细节。
  - StatusDriven 路径只调用 `IRemoteStatusConsumeService`，不直接操作 Merge/删除仓储。
- `RemoteStatusConsumeService`
  - 负责 StatusDriven 主流程编排：读取待处理 → 本地追加 → 可选远端回写。
  - 负责局部异常隔离：单页失败不中断已完成页统计；不可恢复错误抛出给上层。
- `OracleStatusDrivenSourceReader`
  - 只负责读取，不负责业务回写。
  - 输出数据必须包含 `__RowId`，为空时该行进入“不可回写”统计。
- `SqlServerAppendOnlyWriter`
  - 只做批量追加，不做去重、不做更新、不做删除。
- `OracleRemoteStatusWriter`
  - 只按 `ROWID` 更新状态，不接受主键条件更新。

## 10. 配置取值与默认值矩阵
### 10.1 SyncMode
- 可选值：`KeyedMerge`、`StatusDriven`。
- 默认值：未配置时 `KeyedMerge`。
- 校验：非枚举值启动即失败，错误信息中文。

### 10.2 StatusColumnName
- 可选范围：源端真实列名，必须通过安全标识符校验。
- 默认值：`TASKPROCESS`。
- 生效条件：仅 `StatusDriven` 使用。

### 10.3 PendingStatusValue
- 可选范围：字符串或 `null`。
- 默认值：`N`。
- 语义：
  - 非 null：`StatusColumnName = PendingStatusValue`
  - null：`StatusColumnName IS NULL`

### 10.4 CompletedStatusValue
- 可选范围：字符串或 `null`（是否允许 null 需待确认）。
- 默认值：`Y`。
- 生效条件：`ShouldWriteBackRemoteStatus=true` 时使用。

### 10.5 ShouldWriteBackRemoteStatus
- 可选值：`true/false`。
- 默认值：`true`（建议）或按业务确认。
- 语义：关闭时仅本地追加，不执行远端状态更新。

### 10.6 StatusBatchSize
- 可选范围：建议 `1~100000`。
- 默认值：建议 `5000`。
- 生效条件：仅 `StatusDriven` 消费批处理使用。

### 10.7 WriteBackCompletedTimeColumnName
- 可选范围：源端真实列名（仅允许字母、数字、下划线）或 `null`。
- 默认值：`null`（不回写完成时间审计列）。
- 生效条件：`StatusDriven` 且 `ShouldWriteBackRemoteStatus=true`。

### 10.8 WriteBackBatchIdColumnName
- 可选范围：源端真实列名（仅允许字母、数字、下划线）或 `null`。
- 默认值：`null`（不回写批次号审计列）。
- 生效条件：`StatusDriven` 且 `ShouldWriteBackRemoteStatus=true`。

## 11. StatusDriven 时序（逐页）
1. 构建读取条件（等值或 `IS NULL`）。
2. 读取 Oracle 行数据并附带 `__RowId`。
3. 将数据批量追加到 SQL Server 目标表。
4. 若开启回写：仅对具备 `__RowId` 的已追加行按 `ROWID` 批量更新远端状态。
5. 记录页级统计：读取、追加、回写成功、回写失败、因缺少 `__RowId` 跳过回写。
6. 汇总批次统计并回传 `SyncExecutionService`。

## 12. 失败隔离与重试策略（实现约束）
- Oracle 读取异常：记录中文错误日志并按统一隔离器策略重试（使用现有 Polly 能力）。
- 本地追加失败：当前页失败，后续页可按策略中止或继续（待确认后固化）。
- 远端回写失败：
  - 不得影响已完成的本地追加数据回滚（避免跨库分布式事务）。
  - 需输出可追踪日志（含 TableCode、BatchId、PageNo、失败 RowId 数）。
  - 失败项进入可重试通道（后续补偿策略待确认）。
