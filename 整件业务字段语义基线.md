# 整件业务字段语义基线

## 1. 目标
- 固化整件任务字段语义，确保与拆零链路在状态推进与接口契约上保持一致。

## 2. 字段语义（基线）

| 字段名 | 语义 | 阶段 | 说明 |
|---|---|---|---|
| `TaskId` | 本地业务任务标识 | 全链路 | 主定位键。 |
| `Barcode` | 整件条码 | 扫描/格口/回传 | 与任务关联的核心业务键。 |
| `PackageType` | 包装类型 | 扫描/规则 | 区分整件业务路径。 |
| `ScannedAtLocal` | 扫描时间（本地） | 扫描 | 记录扫描事件发生时间。 |
| `TargetChuteCode` | 目标格口编码 | 请求格口 | 返回给外部程序执行分拣。 |
| `DroppedAtLocal` | 落格时间（本地） | 落格回传 | 真实落格后写入。 |
| `TaskStatus` | 业务任务状态 | 全链路 | 与拆零一致，使用统一状态机。 |
| `FailureReason` | 失败原因 | 异常分支 | 用于审计与补偿。 |
| `FeedbackStatus` | 业务回传状态 | 回传/补偿 | 标识回传闭环进度。 |
| `TraceId` | 链路追踪标识 | 全链路 | 便于跨系统追踪。 |

## 3. 状态推进口径
- `Created -> Scanned -> Dropped -> FeedbackPending`
- 回传失败任务进入待补偿路径，保持可重入。

## 4. 约束
- 不将同步自动回写状态视为业务完成。
- 不在 Controller 层直接执行任务状态机跳转。
- 所有时间字段使用本地时间语义。
