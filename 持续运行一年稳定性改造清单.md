## 3. P0（必须优先完成）
### 3.1 运行期内存上限治理
- 问题： 目标端快照常驻内存且随数据增长持续膨胀。
- 改造项：
  - 引入按表惰性加载 + 空闲回收机制。
  - 限制单轮处理与单表缓存上限，超限时降级为分段处理。
  - 增加内存水位监控与告警阈值。
- 验收标准：
  - 长稳压测期间内存曲线可收敛，无持续单调增长。
  - 达到阈值后可触发降级策略，服务不崩溃。

### 3.2 目标快照写入性能与文件体积治理
- 问题： 变更即整表序列化写盘，数据规模上升后 I/O 退化明显。
- 改造项：
  - 优化为增量落盘或分片持久化，减少整表重写。
  - 增加文件大小阈值、轮转与压缩策略。
  - 建立启动自检：磁盘空间、目录权限、文件可读写性。
- 验收标准：
  - 大表场景下单轮耗时显著下降且稳定。
  - 磁盘告警前可主动降级或阻断危险写入。

### 3.3 故障重试与风暴控制（Polly）
- 问题： 固定周期重试在依赖故障时易形成错误风暴。
- 改造项：
  - 对外部依赖接入 Polly：指数退避 + 抖动 + 熔断 + 超时 + 限流。
  - 区分可重试与不可重试异常，避免无效重试。
  - 加入连续失败阈值后的自动降级与告警。
- 验收标准：
  - 外部故障期间错误日志速率可控，不出现雪崩。
  - 依赖恢复后可自动回到正常吞吐。

### 3.4 时间窗口一致性防护（本地时间语义）
- 问题： 跨 DST/系统时钟漂移可能导致窗口重叠或空洞。
- 改造项：
  - 明确窗口边界规则并固化为统一策略。
  - 增加时间回拨/跃迁检测与保护逻辑。
  - 增加时间相关回归用例（含 DST 边界）。
- 验收标准：
  - DST 切换与时钟扰动场景无重复/漏同步。
  - 关键时间策略可审计、可追踪。
