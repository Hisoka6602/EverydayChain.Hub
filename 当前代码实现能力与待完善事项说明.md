# 当前代码实现能力与待完善事项说明

## 1. 当前实现（按主链路）

### 1.1 启动与任务调度
- Host 层通过 `EverydayChain.Hub.Host.Workers.SyncBackgroundWorker` 和 `EverydayChain.Hub.Host.Workers.RetentionBackgroundWorker` 周期性执行同步与保留策略治理。
- 同步调度间隔由 `EverydayChain.Hub.Domain.Options.SyncJobOptions.PollingIntervalSeconds` 配置控制。
- 启动阶段由基础设施扩展完成依赖注册、配置校验、数据库迁移与分表预建。
- 危险操作（如 DDL 建表/删表、批量删除）统一通过 `IDangerZoneExecutor` 执行，具备超时、重试、熔断保护。

### 1.2 增量同步主流程
- `EverydayChain.Hub.Application.Services.SyncOrchestrator` 负责读取启用表配置、计算窗口、按优先级与并发上限调度多表。
- `EverydayChain.Hub.Application.Services.SyncExecutionService` 负责分页读取、暂存、幂等合并、删除识别执行、审计写入与检查点提交。
- 检查点与目标状态采用文件持久化，关键写入路径具备原子替换与磁盘阈值保护。
- 检查点默认路径为应用基目录下 `sync-checkpoints.json`（可通过 `SyncJob.CheckpointFilePath` 覆盖）。

### 1.3 删除与保留期治理
- 删除链路支持策略化执行（禁用/软删/硬删）与 DryRun 审计。
- 保留期链路支持分表识别、过期分表清理、回滚脚本生成与 DryRun。
- 运维脚本覆盖体检、灾难恢复与稳定性演练。

### 1.4 可观测与稳定性
- NLog 输出业务日志并落盘，包含吞吐、失败率、滞后、积压等关键指标。
- 主循环提供表级超时保护与看门狗卡死检测。
- 运行期存储守护支持启动自检、空间阈值阻断与内存水位告警。

## 2. 当前能力总结

- 已具备 Oracle 到 SQL Server 的可运行增量同步骨架，支持多表并发与优先级调度。
- 已具备业务键幂等合并、删除识别治理、检查点断点续跑、批次与审计日志闭环。
- 已具备分表自动预建与分表保留期治理，包含危险动作隔离与回滚能力。
- 已具备稳定性治理基础能力（磁盘/内存守护、超时/重试/熔断、演练脚本）。
- 已具备基础回归测试能力（`EverydayChain.Hub.Tests`）与解决方案级构建测试通路。
