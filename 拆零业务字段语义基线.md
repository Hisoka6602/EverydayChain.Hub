# 拆零业务字段语义基线

## 1. 目标
- 固化拆零任务关键字段语义，确保同步、扫描、落格与回传链路使用一致字段口径。

## 2. 字段语义（基线）

| 字段名 | 语义 | 阶段 | 说明 |
|---|---|---|---|
| `TaskId` | 本地业务任务标识 | 全链路 | 作为任务主定位键。 |
| `Barcode` | 业务条码 | 扫描/格口/回传 | 可作为任务辅助定位键。 |
| `DeviceCode` | 扫描设备编码 | 扫描 | 用于设备侧追踪与审计。 |
| `ScannedAtLocal` | 扫描时间（本地） | 扫描 | 记录扫描发生时间。 |
| `TargetChuteCode` | 目标格口编码 | 请求格口 | 由格口规则计算返回。 |
| `DroppedAtLocal` | 落格时间（本地） | 落格回传 | 仅在真实落格完成后写入。 |
| `TaskStatus` | 任务状态 | 全链路 | 使用 `BusinessTaskStatus` 语义。 |
| `FailureReason` | 失败原因 | 异常分支 | 用于排障与补偿。 |
| `FeedbackStatus` | 业务回传状态 | 回传/补偿 | 标识待回传、已回传或失败待补偿。 |
| `TraceId` | 链路追踪标识 | 全链路 | 用于调用链关联。 |

## 3. 状态推进口径
- `Created -> Scanned -> Dropped -> FeedbackPending`
- 异常路径保留失败原因，禁止隐式吞错。

## 4. 约束
- 扫描阶段不写落格结果字段。
- 请求格口阶段不确认落格状态。
- 落格回传阶段才能推进落格状态与落格时间。
