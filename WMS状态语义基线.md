# WMS状态语义基线

## 1. 目标
- 固化同步状态与业务状态语义，避免“同步自动回写”和“业务回传”混用。
- 为后续 API 与服务实现提供统一判定口径。

## 2. 读取状态（远端读取标识）
- 字段：`TASKPROCESS`
- 数据类型：`CHAR(1)`（固定单字符枚举）
- 枚举语义：
  - `N`：未读取（尚未被同步消费）
  - `Y`：已读取（已被同步消费）

## 3. 回传状态（远端回传标识）
- 字段：`STATUS`
- 数据类型：`CHAR(1)`（固定单字符枚举）
- 枚举语义：
  - `N`：未回传
  - `Y`：已回传

## 4. 自动回写字段（同步链路）
- 定义：Oracle -> 本地同步成功后，可由同步消费链路自动回写的远端字段。
- 典型字段：
  - `TASKPROCESS`
  - `OPENTIME`
- 约束：
  - 仅由同步消费链路执行；
  - 不承载业务“已落格/业务完成”语义；
  - 与业务回传链路严格分离。

## 5. 业务回传字段（业务链路）
- 定义：由外部程序触发业务动作后，本地业务状态推进并回传的字段。
- 典型字段：
  - 已落格状态
  - 落格时间
  - 扫描结果字段
- 约束：
  - 业务回传不能等同于同步自动回写；
  - 必须在“落格回传 API”后推进，不在扫描上传阶段直接完成。

## 6. 业务任务状态基线（本地）
- 字段：`BusinessTask.Status`
- 数据类型：`VARCHAR(32)`（存储枚举项字符串名称，如 `Created`、`Scanned`）
- `Created`：任务已物化，等待业务处理。
- `Scanned`：已完成扫描上传处理。
- `Dropped`：已确认落格成功。
- `FeedbackPending`：已落格，待业务回传。

## 7. 边界与禁区
- 禁止将“同步成功”直接映射为“业务已完成”。
- 禁止将业务回传逻辑放入同步状态消费链路。
- 禁止在 Controller 层直接落业务状态机规则。
